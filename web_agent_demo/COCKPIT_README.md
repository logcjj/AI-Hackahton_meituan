## 3. 关键真值（口径，答辩须背熟）

### 3.1 头牌 `large_seed301`
- **Baseline 对比**：传统**纯贪心**真跑 `2097.658` vs `solver_v4` `657.104`，
  **降本 68.67%**（`cost_pct=68.6744`）。判定为 **严格优于**（`strictly_better=True`）——
  口径是**期望成本更低 + 覆盖不更差**，**不宣称省骑手**（骑手占用为中性记录项）。
  本地平台成本 `657.104` 与官方目标口径一致。
- **最优性证书**：optimality **gap 41.82%**（仅基于 **r1** 这一实例），下界 LB≈`382.29`、
  上界 UB（本解）≈`657.10`。**gap 只用 r1，不外推、不宣称泛化**。
- **自进化（机制真实，对成绩零贡献）**：已晋级 `gen01_M1_003`；策略注册表 **24 条 / 19 采纳**；
  五算子 0 字节克隆（`clones=0`）、SHA256 可复现；机制可独立验证，但**对最终派单成绩零贡献**，
  且为 **stub 无 live LLM**（接真 LLM 一条命令）。

### 3.2 6 场景真值矩阵（`--scene-matrix` 现场真跑，各场景互异）
| 场景 | regime | tasks | greedy_cost | v4_cost | covered(贪心→v4) | gap(r1)% | strictly> |
|---|---|---:|---:|---:|---:|---:|:--:|
| 头牌 large_seed301 | bundle-heavy(官方) | 40 | 2097.658 | 657.104 | 40/40 → 40/40 | 41.82 | True |
| 午高峰爆单 peak | large | 40 | 2746.982 | 1301.231 | 40/40 → 40/40 | 29.62 | True |
| 雨天低接单意愿 rain | low_willing | 30 | 2693.505 | 2017.046 | 30/30 → 30/30 | 28.51 | True |
| 骑手稀缺商圈 scarce | scarce | 30 | ~2.3e3 | 1167.349 | **~20/30 → 30/30** | 50.41 | True |
| 合单机会密集 bundle | bundle_heavy | 24 | ~1.4e3 | 563.869 | 24/24 → 24/24 | 31.73 | True |
| 新店突发订单 newshop | high_noise | 30 | ~2.0e3 | ~1.0e3 | 30/30 → 30/30 | ~43.x | True |

- **只有头牌 `large_seed301` 一行是确定性的**（4 个数字逐次复现、0 漂移）。**5 个场景行的 greedy_cost / v4_cost / gap / 覆盖均为「最近一次实跑快照」，会逐次真跑微动**（如 peak v4_cost 在 ~1298–1312、scarce 贪心覆盖在 ~20–21/30 间浮动）。
- **漂移的真正原因是求解链路自身的非确定性**（`run_blind_solve` 与贪心 `compute_baseline` 的多线程/启发式带来微小浮动）——**不是实例在变**：固定 seed 下 `generate_case` 合成的脱敏样例**字节相同**（SHA256 逐次一致）。这恰是「真跑、非写死」的证据；**口头引用任何非头牌的具体小数前，务必现场跑一次 `--scene-matrix` 以当次真值为准**。
- **各场景 cost / 覆盖 / gap 互异**（6/6 distinct 实例键），证明非同一份数据、非写死输出。
- **scarce 体现覆盖瓶颈**：贪心**欠覆盖约 20/30**、v4 **全覆盖 30/30**——这是反硬编码、秀泛化的最强一行。
- **6/6 上下文 v4 均严格优于贪心（成本口径）**；每个场景 gap 各自基于其 r1，**不跨场景外推**。

## 4. 数据边界（真实 vs 演示合成，绝不混淆）

| | 字段 |
|---|---|
| **真实**（直读 solver_v4 / 感知 / 证书 / 四方 / 进化真实输出 + 纯贪心真跑） | 派单关系、接单意愿 willingness、分数 score、`expected_cost`、感知特征、覆盖率、最优性 gap(**r1**)、5 场景脱敏样例的现场真求解输出、记忆注册表 24/19 |
| **演示合成层**（`seed=20260620`，**非真实 GPS / 财务**） | 地图坐标、距离、ETA、天气、商圈名、金额 |

**6 张 KPI 中：**
- 4 张**真值卡**（绿点高亮）：预计完成率、预计无人接单数、骑手占用数、相对省心基线改善。
- 2 张**演示换算卡**（**金色虚线边框 + 半透斜纹 + 右上「演示」徽标 + 标签前「演示换算」金标**）：
  - **履约成本指数 31.3**（以贪心=100 归一化的展示口径指标）
  - **预计商业收益 2,575,290 ¥/日**（ROI 公式演示换算）

  → 视觉上一眼可分，**绝不可被读成真实业务成本/财务**。
- 5 个业务场景为**脱敏随机样例**（`generate_case` 现场合成，非官方数据、非记忆样例），但**求解全程真跑**、输出全为真值。

## 6. 7 分钟答辩 Demo 流（建议）

1. **首屏钩子（~0.5min）**：打开页面即自动求解，五环点亮、KPI count-up、地图焦点订单组发光。
   一句话点题：「10 秒内对赛题真实数据生成高可靠派单方案」。
2. **盲测一镜到底（~1.5min）**：点 **▶ 重新演示**，让评委看「从骨架到全量真值」的完整自主求解动画；
   强调对真实 `large_seed301` 现场跑、非预录。
3. **切场景秀泛化（~1.5min）**：依次点 2~3 个业务场景（尤其 **骑手稀缺商圈**），看预告小字 → 点击现场合成脱敏样例真 solve；
   配合 `--scene-matrix` 一句话：「6 场景真值互异、scarce 贪心欠覆盖约 20/30 而 v4 全覆盖 30/30」。
4. **Baseline 去硬编码（~1min）**：滚到底部 Baseline 面板，讲头牌 **−68.67%**（纯贪心 2097.658 → 657.104），
   强调「严格优于·成本口径」「真跑非硬编码」。
5. **自进化 + 记忆库（~1.5min）**：点首屏金色角标「查看证据 ↓」展开自进化全面板；再开 `/memory` 记忆页（24/19、`gen01_M1_003`），
   **主动声明「机制真实但对最终成绩零贡献」**（诚实是加分项）。
6. **四方共赢 + 商业 ROI（~0.5min）**：拖 α 滑块看 Pareto 取舍；ROI 模拟器现场调参，讲换算锚定真实 68.67%。
7. **证书收口（~0.5min）**：最优性证书（gap 41.82%·仅 r1·LB382.29/UB657.10）作为「可证明」的技术深度收尾。

## 7. 诚实红线提醒（答辩务必守住）

- **不宣称 706 泛化**：706.197 是整体实力口径，单实例/单口径数字不外推为通用保证。
- **不宣称自进化提分**：自进化机制真实、可验证，但**对最终派单成绩零贡献**；不要说「靠进化把分数提上去」。
  `/memory` 页 held-out Δ 是机制内部留出集指标，非派单成绩。
- **gap 只用 r1**：最优性 gap 仅基于各自的 r1 实例（头牌 41.82%），不跨场景、不跨实例外推。
- **场景真值以实跑为准**：6 场景矩阵数字由 `generate_case` 现场合成真跑产出，本文档表格为最近一次实跑快照；
  口头引用前最好现场跑一次 `--scene-matrix`。
- **演示层必须标注**：地图坐标/距离/ETA/天气/金额、履约成本指数、商业收益均为演示换算（`seed=20260620`，非 GPS/财务），
  界面已用金色虚线卡 + 「演示/演示换算」徽标 + 数据边界带显式标注，口头讲解时也应同步点明。
- **严格优于的边界**：仅指成本口径（期望成本更低 + 覆盖不更差），**不宣称省骑手**。
