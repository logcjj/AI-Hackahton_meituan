# Solver variants v6

## `solver_v6_1_multi_bundle_scarce_reassign.py`

当前主文件同款，基于 V4_1 稳定线继续补两类真实缺口：

1. 支持任意长度合单
   - 不再假设 `task_id_list` 最多只有 2 单或 3 单。
   - 主合单搜索接受所有 `len(task_ids) >= 2` 的候选。
   - 局部搜索支持：
     - 大合单拆成多个单任务；
     - 多个单任务合成任意长度合单；
     - 多个小合单合成更大合单。

2. 稀缺骑手场景补单任务重分配
   - 旧版在 `scarce` 时跳过 `_reassign_single_solution`。
   - 现在把单任务多派解也做最小费用重分配，作为候选参与最终比较。
   - 本地稀缺代理样本从约 `1140.76` 降到约 `1081.39`。

## 本地验证

- `large_seed301`: 约 `657.8`，覆盖 `40/40`，骑手无重复。
- 低意愿代理：无明显变化。
- 稀缺代理：有明显改善。
- 5 单合成样例能输出 `T1,T2,T3,T4,T5`。
- `(T1,T2)+(T3,T4)` 合成样例能输出 `T1,T2,T3,T4`。

## 提交建议

这个版本比 V4_4 风险低：不使用跨 `task_key` 重复派单，只利用输入中合法提供的合单候选。下一次要冲 640 线，优先提交这个版本。
