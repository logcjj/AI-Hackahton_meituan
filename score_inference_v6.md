# v6 反推记录

## 为什么 V4/V5 没明显突破

从平台反馈看，V4_1 的提升主要来自小样本和一点点稀缺样本，`low_willingness_seed501` 仍是 `1799.90`。这说明普通 large/medium 的模型已经基本贴住，平均分差距主要不是这些 case。

按 V4_1 的平台分数估算：

- 除 `low_willingness_seed501` 和 `scarce_couriers_seed401` 外，其余 8 个 case 总和约 `3754.97`。
- 要到 `634.49`，low+scarce 合计要从 `3389.24` 降到约 `2589.93`。
- 要到 `593.32`，low+scarce 合计要降到约 `2178.23`。

所以真正突破必须打 low/scarce。

## 已排除的方向

- 跨 `task_key` 交叉多派：V4_4 平均变差，尤其打坏 `scarce`，不是正路。
- 低意愿简单放大/缩小拒单惩罚：本地代理没有改善，且容易耗时。
- 二合单 rewire：本地未改善，还会影响 large 的稳定性，已从主线移除。

## v6 的有效改动

1. 任意长度合单支持
   - 如果 hidden 的 low/scarce 有 4 单、5 单或更多合单候选，旧版会漏掉。
   - v6 不再限制合单长度。

2. 稀缺场景单任务重分配
   - 旧版 `scarce` 直接跳过单任务重分配。
   - 本地稀缺代理中，单任务解重分配后显著变好，并最终把代理分从约 `1140.76` 降到约 `1081.39`。
