# Solver variants v5

## 已验证反馈

- `V4_1`: 平均 `714.42`，比 715 线只小幅提升。
- `V4_4`: 平均 `727.50`，说明“跨 task_key 交叉多派”不是有效方向，尤其会打坏 `scarce_couriers_seed401`。

## v5 文件

1. `solver_v5_1_pair_rewire_strict.py`
   - 严格合法约束内增加二合单重组搜索。
   - 本地样例基本不变，作为安全实验。

2. `solver_v5_2_low_penalty2.py`
   - 低意愿场景把 score 缩小为 `0.5` 来生成候选，等价于更重视拒单惩罚。
   - 本地低意愿代理更差，不优先推荐。

3. `solver_v5_3_low_penalty3.py`
   - `v5_2` 更激进版本，score 缩小为 `0.33`。
   - 本地低意愿代理更差，不优先推荐。

4. `solver_v5_4_low_penalty_half.py`
   - 反方向实验，低意愿场景把 score 放大为 `2.0`。
   - 本地代理没有明显变化，不优先推荐。

5. `solver_v5_5_triple_bundle.py`
   - 初步支持三单合单。

6. `solver_v5_6_multi_bundle.py`
   - 当前主文件同款，支持输入中任意长度的合单候选。
   - 不再把核心合单搜索写死为 `len(task_ids) == 2`。
   - 合单拆分、多个单任务合并、已有小合单合并为更大合单，都支持任意长度合单。
   - 对 `large_seed301` 无变化：本地约 `657.808`，约 `7.6s`。
   - 合成 5 单样例能正确选择 `T1,T2,T3,T4,T5`。
   - 合成 `(T1,T2)+(T3,T4)` 样例能正确替换为 `T1,T2,T3,T4`。

## 当前建议

如果下一次提交要测“hidden 是否存在三单/多单合单”，提交 `solver_v5_6_multi_bundle.py`。它比 V4_4 风险低很多，因为它不赌非法交叉多派，只是不再忽略官方输入里可能存在的多订单合单候选。
